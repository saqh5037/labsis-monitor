Conexiones a la Base de Datos
Verde = trabajando activamente. Azul = disponibles esperando (normal). Rojo = transacciones abandonadas que bloquean la base de datos. El rojo siempre debe ser 0.
Eficiencia de Memoria de la Base de Datos
Qué tan rápido responde la base de datos. Lo ideal es 99% o más. Si baja de 95%, la base de datos tiene que buscar información en el disco duro en vez de la memoria, y eso es 100 veces más lento.
Operaciones Lentas en la Base de Datos
Barras rojas = cantidad de operaciones que tardan más de 30 segundos (ideal: 0). Área amarilla = cuánto tiempo lleva la operación más lenta. Si la línea amarilla sube sin parar, hay una transacción "zombie" atorada que necesita ser terminada.
Bloqueos en la Base de Datos
Barras rojas = operaciones que están atoradas esperando a que otra termine. Barras amarillas = conexiones que la app abrió y no cerró ("transacciones abandonadas"). Ambos deben estar en 0.
Velocidad de la Base de Datos
Verde = operaciones completadas con éxito por segundo. Rojo = operaciones que fallaron. Si la línea roja sube, hay errores en el sistema que están afectando a los usuarios.
Conexiones de Servidores a la Base de Datos
Cuántas conexiones tiene cada servidor hacia la base de datos. Si un servidor acapara muchas, puede dejar al otro sin espacio para trabajar. El límite recomendado es 60 por servidor.
Deadlocks (Conflictos de Transacciones)
Cuántas veces dos procesos se bloquearon mutuamente y PostgreSQL tuvo que matar uno. Lo ideal es 0. Si sube, hay bugs de concurrencia en la aplicación que el equipo de desarrollo debe corregir.
Archivos Temporales de BD
Cuando un query necesita más memoria de la asignada (work_mem), genera archivos temporales en disco. Esto es mucho más lento. Si sube, hay queries que necesitan optimización o más work_mem.
Mapa de Calor — Patrones de Carga por Hora y Día
Muestra en qué horarios y días la base de datos tiene más carga. Útil para identificar ventanas de mantenimiento y patrones recurrentes. Rojo = alta carga, Verde = baja carga.
Carga General del Sistema
Qué tan saturados están los servidores. Los servidores tienen capacidad para 8 tareas simultáneas. Si la línea pasa de 8, el servidor está sobrecargado. Si pasa de 16, es una saturación severa.
Espacio en Disco
Cuánto espacio queda en los discos de cada servidor. Verde = bien. Amarillo = queda poco. Rojo = urgente, si se llena el disco el servidor se detiene por completo.
Actividad de Disco (I/O)
Cuántos datos leen y escriben los servidores en disco. Picos altos pueden indicar: backups en ejecución, queries pesados que leen mucha data, o logs creciendo demasiado.
Tráfico de Red
Cuántos datos entran y salen de los servidores por la red. Picos anormales pueden indicar: exports masivos de datos, sincronización, o tráfico inusual.
Timeline de Eventos
Registro cronológico de todos los eventos detectados automáticamente. Útil para post-mortem y entender qué pasó.
